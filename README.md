# Token RugCheck MCP

Solana token safety audit for AI Agents â€” detect Rug Pulls before your agent buys. Powered by ag402 micropayments.

## What It Does

Input a Solana token contract address, get a three-layer safety audit report:

- **Action Layer** â€” machine-readable verdict (`is_safe`, `risk_score`, `risk_level`)
- **Analysis Layer** â€” LLM-friendly risk summary with red/green flags
- **Evidence Layer** â€” raw data for human cross-verification

Data sources: RugCheck.xyz + DexScreener + GoPlus Security (concurrent fetch, graceful degradation).

## API Reference

### Endpoints

| Method | Path | Rate Limit | Description |
|--------|------|------------|-------------|
| GET | `/audit/{mint_address}` | 60/min per IP | Full safety audit report |
| GET | `/health` | Unlimited | Service health + upstream status |
| GET | `/stats` | 10/min per IP | Request counts + cache hit rate |

Rate-limited requests return `429 Too Many Requests` with a `Retry-After` header.

### Health Check

```
GET /health
```

| `status` | Meaning | Action |
|----------|---------|--------|
| `ok` | All systems normal | â€” |
| `degraded` | No successful upstream call in 2 minutes | Check network / upstream APIs |

```json
{

  "chain": "solana",
  "action": {
```

### `metadata` Fields

| Field | Type | Description |
|-------|------|-------------|
| `data_sources` | `string[]` | Which upstream APIs returned data |
| `data_completeness` | `string` | `full` / `partial` / `minimal` |
| `cache_hit` | `bool` | Whether this response came from cache |
| `data_age_seconds` | `int\|null` | `0` = fresh fetch, `>0` = seconds since original fetch |
| `response_time_ms` | `int` | Server-side processing time |

### Security

- **Rate limiting** â€” per-IP sliding window on `/audit` (60/min) and `/stats` (10/min)
- **Upstream protection** â€” `asyncio.Semaphore(20)` caps concurrent outbound API calls
- **Error sanitization** â€” exceptions return categorized codes, never internal paths
- **Cache safety** â€” all cache operations are `asyncio.Lock`-guarded

### Design Notes

- **Cache TTL 30s** â€” for on-chain security data, freshness beats speed
- **Liquidity exemption** â€” tokens with >= $1M liquidity are exempt from LP-protection and holder-concentration rules, preventing false positives on BONK/WIF/JUP
- **Single worker** â€” in-memory cache is per-process; use `--workers 1` (default) or switch to Redis for multi-worker
